<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> middleware | Unauthenticated/forged session cookies cross here on every portal request |
| Browser -> Server Action | Login credentials cross here |
| Server Component -> Postgres (RLS) | Role data read here; client must not be able to read other users' roles |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Spoofing | middleware.ts session refresh | mitigate | updateSession() uses getUser() (re-validates JWT with Auth server), never getSession() (cookie-only) |
| T-04-02 | Spoofing | loginAction | mitigate | Auth handled by Supabase signInWithPassword (hashing, rate limiting); no custom credential logic |
| T-04-06 | Information Disclosure | profiles RLS | mitigate | SELECT policy USING (auth.uid() = user_id) — anon key cannot read other users' role rows |
| T-04-07 | Elevation of Privilege | role storage | mitigate | Role stored in profiles table (server-side DB), not JWT app_metadata client could forge (D-01) |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this phase (RESEARCH.md Package Legitimacy Audit: no new packages) |
</threat_model>
