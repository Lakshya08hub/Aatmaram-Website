<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> /login page | Credentials entered and submitted here by anonymous users |
| LoginForm -> loginAction (Server Action) | Credentials cross to server; Server Actions carry built-in CSRF/origin checks |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-09 | Tampering | Login form submission | mitigate | Next.js Server Action (loginAction) includes built-in origin validation/CSRF |
| T-04-10 | Spoofing | Credential check | mitigate | Supabase signInWithPassword (hashing + built-in rate limiting); no custom auth |
| T-04-02 | Information Disclosure | Error messages | accept | Supabase returns generic invalid-credentials message; no user-enumeration detail surfaced |
| T-04-11 | Auth bypass | Login redirect loop | mitigate | /login placed at app/login/, OUTSIDE (portal) guard (Pitfall 2); page itself redirects authed users to /dashboard (D-12) |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this phase |
</threat_model>
