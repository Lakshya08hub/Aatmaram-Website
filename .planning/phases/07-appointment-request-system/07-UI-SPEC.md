## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label / Heading (page) / Table header / Badge text | 14px–28px | 600 (semibold) | 1.2–1.4 |

Exact role mapping:

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label (form) | 14px | 600 (semibold) | 1.4 |
| Heading (page) | 28px | 600 (semibold) | 1.2 |
| Table header | 12px | 600 (semibold) | 1.4 |
| Badge text | 12px | 600 (semibold) | 1.4 |

Notes:
- shadcn FormLabel defaults to 500 (medium) — do not override with a global rule; 500 passes through as a shadcn default and is not part of this typography contract.
- Portal tab labels (Pending / Contacted / Confirmed / Cancelled) use 14px/600 (semibold).
- All headings use `font-semibold` (600). Do not use `font-bold` (700) anywhere in this phase.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F8FAFC` (`--color-bg`) | Page background, table row backgrounds |
| Secondary (30%) | `#FFFFFF` (`--color-surface`) | Card surfaces, Sheet panel, Dialog background |
| Accent (10%) | `#1E40AF` (`--color-primary`) | Submit CTA button, active tab indicator, focused ring |
| Destructive | shadcn `--destructive` (red) | Cancel appointment action button only |

Accent reserved for: submit button (`bg-blue-800 hover:bg-blue-900`), active Tabs trigger underline/highlight, focused input ring. NOT used for table row hover, badge colors, or secondary actions.

Status badge colors (semantic, not accent):
- Pending: `bg-yellow-100 text-yellow-800` (amber — awaiting action)
- Contacted: `bg-blue-100 text-blue-800` (informational)
- Confirmed: `bg-green-100 text-green-800` (success)
- Cancelled: `bg-red-100 text-red-800` (terminal/destructive)

Source: existing project tokens in `app/globals.css`; blue-800/blue-900 from AppointmentForm.tsx line 212.

## Visuals

### Public Appointment Form — Focal Point

The "Request Appointment" submit button is the primary focal point of the public appointment form screen: full-width, accent color (`#1E40AF`, `bg-blue-800 hover:bg-blue-900`), size `lg`, minimum height 44px. No other element on the form should compete with it visually.

### Portal AppointmentsClient

Structural layout mirrors StaffClient.tsx exactly: page heading (`font-semibold` 28px) at top, Tabs row below, Table inside each TabsContent. No decorative illustrations. Status badges provide the only color variation within the table.

## Component Inventory

All components are already installed via shadcn. No new installs required.

### Public Form — AppointmentForm.tsx additions

| Component | Usage | Source |
|-----------|-------|--------|
| `Select` + `SelectItem` | preferredTime dropdown (Morning/Afternoon/Evening OPD) | already used for preferredDoctor |
| hCaptcha widget | `@hcaptcha/react-hcaptcha` npm package | new dependency |
| `Button` size="lg" | Submit CTA — existing pattern preserved | shadcn |

preferredTime field placement: after `preferredDate` field, before `reason` field.

preferredTime Select options (exact copy):
1. `"Morning OPD (9am–12pm)"`
2. `"Afternoon OPD (12–3pm)"`
3. `"Evening OPD (3–6pm)"`

hCaptcha widget placement: between the `reason` Textarea and the Submit button. Renders the hCaptcha challenge inline. On verification, stores token in component state. Submit is disabled until token is present.

### Portal — AppointmentsClient component

Replicate StaffClient.tsx structural pattern exactly:

| Component | Usage |
|-----------|-------|
| `Tabs` / `TabsList` / `TabsTrigger` / `TabsContent` | 4 tabs: Pending / Contacted / Confirmed / Cancelled |
| `Table` / `TableHeader` / `TableBody` / `TableRow` / `TableCell` | Appointment rows per tab |
| `Badge` | Status display in each row (color per status badge map above) |
| `Select` | Per-row status update dropdown (any→any transitions) |
| `Textarea` | Optional callback notes — appears inline below status Select when status is changed |
| `Button` variant="default" size="sm" | "Save Status" on per-row status update |
| `toast()` (Sonner) + `router.refresh()` | Post-mutation feedback — existing pattern |

No Sheet or AlertDialog needed: status changes are inline in the table row (no full-page edit, no destructive confirm dialog required — Cancelled status is reversible per D-08 any→any transitions).

## Interaction Contracts

### Public Form

| Interaction | Behavior |
|-------------|----------|
| Doctor dropdown | Populated server-side via `getDoctors()`, passed as props. First option is "No Preference" (value: "no-preference"). |
| preferredDate | HTML `date` input, `min` = today's date. Past dates rejected by Zod + native browser. |
| preferredTime | Required Select. No default value — user must choose. Shows FormMessage on submit if empty. |
| hCaptcha | Renders after reason field. Submit Button is `disabled` until hCaptcha token is in state. On form reset after success, `hcaptchaRef.current?.resetCaptcha()` is called. |
| Submit loading state | Button shows `<Loader2 className="animate-spin" />` while Server Action is pending (use `useTransition` or `form.formState.isSubmitting`). |
| Success | `toast.success()` with i18n copy. Form resets. hCaptcha resets. |
| Server error | `toast.error()` with i18n copy describing the problem and next step. |
| Validation errors | Inline `FormMessage` below each field — existing pattern, no change. |

### Portal — AppointmentsClient

| Interaction | Behavior |
|-------------|----------|
| Tab switch | Filters table to show only rows matching that status. Count badge on each tab showing number of records (e.g. "Pending (3)"). |
| Empty tab | Shows empty state: heading + body copy (see Copywriting Contract). |
| Status Select per row | Selecting a new status reveals a `Textarea` below for optional callback notes. "Save Status" button becomes active. |
| Save Status action | Calls Server Action → `toast.success()` → `router.refresh()` → row moves to correct tab. |
| Notes field | Always optional. Placeholder: "Callback notes (optional)". Max 500 chars. |
| Row layout | Columns: Patient Name | Phone | Preferred Doctor | Date + Time | Reason | Status | Actions |
