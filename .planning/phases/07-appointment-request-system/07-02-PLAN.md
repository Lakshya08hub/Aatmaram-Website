# Plan 07-02: Portal Appointments Management

## Goal

Replace the `app/(portal)/appointments/page.tsx` stub with a fully functional Server Component + Client component
pattern (mirroring the Phase 6 staff page). Receptionists and Admins can view all appointment requests in a
4-tab interface and update status with optional callback notes.

This plan depends on 07-01 (the `appointment_requests` table must exist before the DB query runs).

---

## Tasks

### Wave 0 — DB Query Utility (no prior dependencies within this plan)

- [ ] APMT-01: Create `lib/db/appointment_requests.ts` — `AppointmentRequest` type + `getAppointmentRequests()` query
  - File: `lib/db/appointment_requests.ts`
  - Pattern source: `lib/db/staff.ts` — exact same structure, swap entity names.
  - Import `createClient` from `@/lib/supabase/server`.

  **Type definition to export:**
  ```
  export interface AppointmentRequest {
    id: string
    patient_name: string
    phone: string
    preferred_doctor: string
    preferred_date: string       // DATE stored as ISO string
    preferred_time: string       // 'morning' | 'afternoon' | 'evening'
    reason: string
    status: 'pending' | 'contacted' | 'confirmed' | 'cancelled'
    notes: string | null
    created_at: string
  }
  ```

  **Function to export:**
  ```
  export async function getAppointmentRequests(): Promise<AppointmentRequest[]>
  ```
  Implementation: `createClient()` → `supabase.from('appointment_requests').select('*').order('created_at', { ascending: false })`.
  If Supabase returns an error, throw `new Error('Failed to fetch appointments: ' + error.message)`.
  Return `(data ?? []) as AppointmentRequest[]`.

  No admin client needed here — the portal page is behind auth and the Supabase server client uses the session cookie.

  **Verification:** `npx tsc --noEmit` passes on this file.

### Wave 1 — Server Action (depends on: APMT-01 type exported)

- [ ] APMT-02: Create `app/(portal)/actions/appointments.ts` — `requireAppointmentRole` guard + `updateAppointmentStatusAction`
  - File: `app/(portal)/actions/appointments.ts`
  - Mark `'use server'` at top.
  - Import `redirect` from `next/navigation`, `createClient` from `@/lib/supabase/server`.
  - Import `StaffRole` from `@/lib/portal/roles` (or wherever the type is declared — check `lib/portal/roles.ts`).

  **Private helper `requireAppointmentRole` (not exported):**

  Allowed roles array: `const APPOINTMENT_ROLES: StaffRole[] = ['super_admin', 'admin', 'receptionist']`

  Implementation:
  1. `const supabase = await createClient()`
  2. `const { data: { user } } = await supabase.auth.getUser()` — if no user, `redirect('/login')`
  3. Query `profiles` table: `.select('role').eq('user_id', user.id).single()`
  4. If error, no profile, or `!APPOINTMENT_ROLES.includes(profile.role as StaffRole)` → `throw new Error('Forbidden')`
  5. Return `supabase`

  This is a new helper — do NOT reuse `requireAdminRole()` from `staff.ts`. That function only allows `['super_admin', 'admin']` and would block receptionists (D-10).

  **Exported action `updateAppointmentStatusAction`:**

  Signature: `(id: string, status: 'pending' | 'contacted' | 'confirmed' | 'cancelled', notes?: string): Promise<{ error?: string }>`

  Implementation:
  1. Call `const supabase = await requireAppointmentRole()`
  2. `.from('appointment_requests').update({ status, notes: notes ?? null }).eq('id', id)`
  3. If Supabase returns error, throw it.
  4. Return `{}`
  5. Catch block returns `{ error: err instanceof Error ? err.message : 'Unknown error' }`

  Do NOT call `revalidatePath()` — the client component uses `router.refresh()` after the action resolves (consistent with StaffClient pattern).

  **Verification:** `npx tsc --noEmit` passes. Confirm `receptionist` is in the `StaffRole` union type in `lib/portal/roles.ts` before completing this task — if the type does not include `'receptionist'`, add it to the union there and note it in the summary.

### Wave 2 — Portal UI (depends on: APMT-01 type + APMT-02 action)

- [ ] APMT-03: Create `components/portal/AppointmentsClient.tsx` — 4-tab table with inline status update
  - File: `components/portal/AppointmentsClient.tsx`
  - Pattern source: `app/(portal)/staff/StaffClient.tsx` — mirror the Tabs + Table structure. Read that file before writing this one.
  - Mark `'use client'` at top.

  **Imports needed:**
  - `useState` from `react`
  - `useRouter` from `next/navigation`
  - `toast` from `sonner`
  - `AppointmentRequest` from `@/lib/db/appointment_requests`
  - `updateAppointmentStatusAction` from `@/app/(portal)/actions/appointments`
  - shadcn: `Tabs`, `TabsList`, `TabsTrigger`, `TabsContent`, `Table`, `TableHeader`, `TableBody`, `TableRow`, `TableCell`, `TableHead`, `Badge`, `Select`, `SelectContent`, `SelectItem`, `SelectTrigger`, `SelectValue`, `Textarea`, `Button`

  **Component props interface:**
  ```
  interface Props {
    appointments: AppointmentRequest[]
    fetchError?: boolean
  }
  ```

  **Status filtering (top of component body):**
  ```
  const pending   = appointments.filter(a => a.status === 'pending')
  const contacted = appointments.filter(a => a.status === 'contacted')
  const confirmed = appointments.filter(a => a.status === 'confirmed')
  const cancelled = appointments.filter(a => a.status === 'cancelled')
  ```

  **Per-row state pattern:** Use a `Map<string, { status: string; notes: string; dirty: boolean }>` in `useState` to track in-flight edits per row (keyed by appointment id). Initialize lazily on first interaction. This avoids lifting state up and keeps each row independent.

  **`handleSaveStatus(id: string)` async function:**
  1. Read current row state from the map.
  2. Call `const result = await updateAppointmentStatusAction(id, rowState.status as ..., rowState.notes || undefined)`
  3. If `result.error`: `toast.error(result.error)`
  4. Else: `toast.success('Status updated')` then `router.refresh()`
  5. Set `dirty: false` for the row in the map.
  6. Wrap in try/catch — catch calls `toast.error('Failed to update. Try again.')`

  **Tab structure:**
  ```
  <Tabs defaultValue="pending" className="mt-6">
    <TabsList>
      <TabsTrigger value="pending">
        Pending
        {pending.length > 0 && <span className="ml-2 rounded-full bg-amber-100 px-2 py-0.5 text-xs font-semibold text-amber-700">{pending.length}</span>}
      </TabsTrigger>
      <TabsTrigger value="contacted">Contacted {contacted.length > 0 && <span ...>{contacted.length}</span>}</TabsTrigger>
      <TabsTrigger value="confirmed">Confirmed</TabsTrigger>
      <TabsTrigger value="cancelled">Cancelled</TabsTrigger>
    </TabsList>
    <TabsContent value="pending"><AppointmentTable rows={pending} ... /></TabsContent>
    <TabsContent value="contacted"><AppointmentTable rows={contacted} ... /></TabsContent>
    <TabsContent value="confirmed"><AppointmentTable rows={confirmed} ... /></TabsContent>
    <TabsContent value="cancelled"><AppointmentTable rows={cancelled} ... /></TabsContent>
  </Tabs>
  ```

  **`AppointmentTable` inner component (or inline):** Renders a `<Table>` with these columns:
  Patient Name | Phone | Preferred Doctor | Date + Time | Reason | Actions

  Column details:
  - Phone: render as `<a href={`tel:${row.phone}`} className="text-blue-600 underline">{row.phone}</a>`
  - Date + Time: format date as `new Date(row.preferred_date).toLocaleDateString('en-IN')`. Map preferred_time: `morning` → `"Morning OPD (9am–12pm)"`, `afternoon` → `"Afternoon OPD (12–3pm)"`, `evening` → `"Evening OPD (3–6pm)"`. Render both on two lines or separated by a bullet.
  - Reason: truncate to 60 chars with `title={row.reason}` on the cell for full text on hover.
  - Actions column: contains the status Select + notes Textarea + Save button (see below).

  **Actions cell per row (inline — no Sheet required per UI-SPEC):**
  ```
  <div className="flex flex-col gap-2 min-w-[200px]">
    <Select
      value={rowState?.status ?? row.status}
      onValueChange={(val) => updateRowState(row.id, { status: val, dirty: true })}
    >
      <SelectTrigger><SelectValue /></SelectTrigger>
      <SelectContent>
        <SelectItem value="pending">Pending</SelectItem>
        <SelectItem value="contacted">Contacted</SelectItem>
        <SelectItem value="confirmed">Confirmed</SelectItem>
        <SelectItem value="cancelled">Cancelled</SelectItem>
      </SelectContent>
    </Select>
    {rowState?.dirty && (
      <>
        <Textarea
          placeholder="Callback notes (optional)"
          maxLength={500}
          value={rowState.notes ?? ''}
          onChange={(e) => updateRowState(row.id, { notes: e.target.value })}
          className="text-sm"
          rows={2}
        />
        <Button size="sm" onClick={() => handleSaveStatus(row.id)}>Save Status</Button>
      </>
    )}
  </div>
  ```

  **Status badge** (shown in the row alongside the Select, or in a Status column):
  - `pending`: `<Badge className="bg-amber-100 text-amber-800 font-semibold">Pending</Badge>`
  - `contacted`: `<Badge className="bg-blue-100 text-blue-800 font-semibold">Contacted</Badge>`
  - `confirmed`: `<Badge className="bg-green-100 text-green-800 font-semibold">Confirmed</Badge>`
  - `cancelled`: `<Badge className="bg-red-100 text-red-800 font-semibold">Cancelled</Badge>`
  Use `font-semibold` (600) on badge text. Do NOT use `font-bold` anywhere in this component (UI-SPEC constraint).

  **Empty tab state:** When `rows.length === 0`, show:
  ```
  <div className="py-12 text-center text-sm text-muted-foreground">
    <p className="font-semibold">No {tabLabel} appointments</p>
    <p className="mt-1">
      {tabValue === 'pending'
        ? 'New requests will appear here when patients submit the appointment form.'
        : 'No appointments in this status.'}
    </p>
  </div>
  ```

  **Error banner** (when `fetchError` prop is true, shown above the tabs):
  ```
  <div className="mt-4 rounded-md border border-red-200 bg-red-50 px-4 py-3 text-sm text-red-700">
    Could not load appointment requests. Check your connection and refresh.
  </div>
  ```

  **Page heading** (inside the component, above the Tabs):
  `<h1 className="text-2xl font-semibold">Appointment Requests</h1>` — font-semibold (600), not font-bold.

  **Verification:** `npx tsc --noEmit` passes. No `font-bold` class anywhere in the file (use `grep 'font-bold' components/portal/AppointmentsClient.tsx` — should return no matches).

- [ ] APMT-04: Replace `app/(portal)/appointments/page.tsx` stub with Server Component data fetcher
  - File: `app/(portal)/appointments/page.tsx`
  - Read the current stub file first — it contains a "Coming in Phase 7" placeholder.
  - Overwrite it entirely with:

  ```
  import { getAppointmentRequests, AppointmentRequest } from '@/lib/db/appointment_requests'
  import AppointmentsClient from '@/components/portal/AppointmentsClient'

  export const dynamic = 'force-dynamic'

  export default async function AppointmentsPage() {
    let appointments: AppointmentRequest[] = []
    let fetchError = false

    try {
      appointments = await getAppointmentRequests()
    } catch {
      fetchError = true
    }

    return <AppointmentsClient appointments={appointments} fetchError={fetchError} />
  }
  ```

  `export const dynamic = 'force-dynamic'` is mandatory — appointment list changes on every public form submission and must not be cached.

  The portal layout already handles auth guard — no role check needed in the page itself.
