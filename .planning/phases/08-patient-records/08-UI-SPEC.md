## Spacing Scale

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), inline badge padding |
| sm | 8px | Table cell padding, form field gap (`space-y-1`) |
| md | 16px | Default element spacing, form field gap (`space-y-4`) |
| lg | 24px | Section padding, tab margin (`mt-6`) |

| 3xl | 64px | Not used in this phase |

## Color

Inherits portal color contract established in prior phases. No new tokens introduced.

| Role | Tailwind Class | Usage |
|------|----------------|-------|
| Dominant (60%) | `bg-slate-50` | Page background (`main` in portal layout) |

| Sidebar | `bg-slate-900` | Portal sidebar (inherited, no change) |

Accent reserved for: the "Add Patient" button in the page header. All other interactive elements use `variant="ghost"` (icon buttons in table rows) or `variant="outline"` (dismiss buttons in sheet footer).

Semantic status colors used in badges (matching existing portal badge convention):
- Doctor-assigned: `variant="secondary"` (neutral-100 bg, neutral-800 text)
- No doctor assigned: `variant="outline"` (border-only neutral)
- Clinical notes present: `variant="secondary"` (existing pattern)

## Page Layout: `/patients`

### Shell
Matches staff page layout exactly:
```
<div className="p-8">
  {/* Header row */}
  <div className="flex items-center justify-between">
    <h1 className="text-xl font-semibold text-slate-800">Patient Records</h1>
    {/* Primary focal point: the "Add Patient" button in the page header */}
    {/* Visible for receptionist/admin/super_admin; hidden for doctor role */}
    <Button>Add Patient</Button>
  </div>

  {/* Search input — full width below header, mt-4 */}
  <div className="mt-4">
    <Input placeholder="Search by name or phone…" />
  </div>

  {/* Patient table — mt-4, rounded-lg border border-slate-200 bg-white shadow-sm overflow-hidden */}
  <div className="mt-4 rounded-lg border border-slate-200 bg-white shadow-sm overflow-hidden">
    <Table>…</Table>
  </div>
</div>
```

### Role-based "Add Patient" button visibility
- Receptionist: visible
- Admin / Super Admin: visible
- Doctor: button is hidden entirely (doctors do not create records)

### Search Input Behavior
- Type: `Input` (shadcn), placeholder: "Search by name or phone…"
- Placement: Below the header row, `mt-4`, full-width (`w-full`)
- Behavior: Client-side filter, no debounce required — instant filter on each keystroke
- Filters on: `patient_name` (case-insensitive substring) OR `phone` (substring match)
- Clear: No explicit clear button — user clears with keyboard

## Patient Table

### Columns

| # | Column | Doctor view | Receptionist / Admin view |
|---|--------|-------------|--------------------------|
| 1 | Patient Name | shown | shown |
| 2 | Age | shown | shown |
| 3 | Phone | shown | shown |
| 4 | Reason | shown (truncated to 40 chars with ellipsis) | shown (truncated) |
| 5 | Assigned Doctor | shown | shown |
| 6 | Visit Date | shown | shown |
| 7 | Notes | shown as badge: "Added" / "—" | shown as badge: "Added" / "—" |
| 8 | Actions | "Add Notes" icon button (Pencil) | "Edit" icon button (Pencil) |

Notes column: renders a `Badge variant="secondary"` with text "Added" if `clinical_notes` is non-null and non-empty; otherwise a `—` text cell (`text-slate-400`). Do not display raw clinical note text in the table.

Actions column: single icon button (`Pencil`, `h-8 w-8`, `variant="ghost"`), `aria-label` context-dependent:
- Receptionist / Admin: `aria-label="Edit {patient_name}"`
- Doctor: `aria-label="Add notes for {patient_name}"`

Clicking the action button opens the Edit Sheet for that record.

### Sort order
Newest first (`visit_date DESC`, then `created_at DESC` as tiebreaker). Static sort — no sortable column headers in v1.

### Empty state (no records at all)
```
<div className="mt-8 flex flex-col items-center gap-2 text-center">
  <p className="text-sm text-slate-500">No patient records yet.</p>
  <p className="text-xs text-slate-400">Add a new patient record to get started.</p>
</div>
```

### Empty state (search returns no results)
```
<div className="mt-8 flex flex-col items-center gap-2 text-center">
  <p className="text-sm text-slate-500">No records match your search.</p>
  <p className="text-xs text-slate-400">Try a different name or phone number.</p>
</div>
```

## Add / Edit Sheet

Uses `Sheet` (slides from right), `sm:max-w-lg overflow-y-auto` — identical to StaffClient edit sheet.

### Sheet title
- Add flow: "Add Patient Record"
- Edit flow (receptionist/admin): "Edit Patient Record"
- Edit flow (doctor — clinical notes): "Add Clinical Notes"

### Form fields by role

#### Receptionist / Admin / Super Admin — Add and Edit

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| Patient Name | `Input` | Yes | `placeholder="Full name"` |
| Age | `Input type="number"` | Yes | `placeholder="Age in years"`, min=0 max=120 |
| Phone | `Input` | Yes | `placeholder="+91 98765 43210"` |
| Reason for Visit | `Input` | Yes | `placeholder="Chief complaint or reason"` |
| Assigned Doctor | `Select` (shadcn) | No | Populates from `getDoctors()`. Option label: `"Dr. {name} — {specialization}"`. Includes a "None / Unassigned" option as first item. |
| Visit Date | `Input type="date"` | Yes | Defaults to today's date on Add |
| Clinical Notes | Hidden entirely | — | Not shown, not sent |

#### Doctor — Edit (clinical notes only)

All fields above are rendered as **read-only display text** (not form inputs), inside a `space-y-3` container using `<p className="text-sm text-slate-700">` with a `<label className="text-xs font-semibold text-slate-400 uppercase tracking-wide">` above each.

Only one editable field is shown:

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| Clinical Notes | `Textarea` | No | `placeholder="Medications prescribed, treatment notes, follow-up instructions…"`, `rows={6}`, `className="resize-none"` |

A helper note below the textarea: `<p className="text-xs text-slate-400">Only you can edit this field. Changes save immediately on Submit.</p>`

### Read-only field rendering (doctor view)

Each read-only field uses this pattern:
```tsx
<div className="space-y-1">
  <p className="text-xs font-semibold text-slate-400 uppercase tracking-wide">{label}</p>
  <p className="text-sm text-slate-700">{value || '—'}</p>
</div>
```

Wrap all read-only fields in: `<div className="space-y-3 rounded-md border border-slate-100 bg-slate-50 px-4 py-3">`

### Sheet footer buttons

| Role | Dismiss | Submit |
|------|---------|--------|
| Receptionist / Admin (Add) | `variant="outline"` "Discard" | `variant="default"` "Add Patient" |
| Receptionist / Admin (Edit) | `variant="outline"` "Discard Changes" | `variant="default"` "Save Changes" |
| Doctor (clinical notes) | `variant="outline"` "Close" | `variant="default"` "Save Notes" |

Loading state: replace submit button content with `<Loader2 className="mr-2 h-4 w-4 animate-spin" />` + in-progress label ("Adding…" / "Saving…").

### Validation error display

Inline below each field:
```tsx
<p className="text-xs text-red-500">{error.message}</p>
```

## Toast Messages (Sonner)

| Action | Toast type | Message |
|--------|------------|---------|
| Patient record created | `toast.success` | "Patient record added" |
| Patient record updated (receptionist) | `toast.success` | "Record updated" |
| Clinical notes saved (doctor) | `toast.success` | "Notes saved" |
| Any server action error | `toast.error` | Error message from server action (pass through) |
| Fetch error on page load | Inline error banner (not toast) | See Fetch Error Banner below |

### Fetch Error Banner (inline, not toast)
```tsx
<div className="mt-4 rounded-md border border-red-200 bg-red-50 px-4 py-3 text-sm text-red-700">
  Could not load patient records. Check your connection and refresh.
</div>
```
