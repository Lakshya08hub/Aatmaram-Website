    Run npm run dev, then perform ALL five tests below. Mark each as PASS or FAIL.

    TEST 1 — I18N-01: Hindi content completeness
    1. Visit http://localhost:3000/hi
    2. Verify: nav links show होम, हमारे बारे में, विभाग, हमारे डॉक्टर, सेवाएं, संपर्क
    3. Visit http://localhost:3000/hi/about — verify page heading is Hindi (not "About Us")
    4. Visit http://localhost:3000/hi/appointment — verify form labels are Hindi (रोगी का नाम, फ़ोन नंबर, etc.)
    5. Check: NO text "[missing" visible anywhere on any Hindi page
    PASS if: all visible strings are Hindi (except proper nouns: Ayushman Bharat, PM-JAY, ICU, OPD)
    FAIL if: any English placeholder or [missing key] visible

    TEST 2 — I18N-02: Browser language auto-detection (Hindi preference → Hindi site)
    1. Open Chrome DevTools → More tools → Sensors → Locale → set to "hi-IN"
    2. Clear cookies: DevTools → Application → Cookies → Clear all for localhost
    3. Visit http://localhost:3000 (no path)
    4. Verify: browser redirected to http://localhost:3000/hi (or http://localhost:3000/hi/)
    5. Verify: page content is in Hindi
    PASS if: auto-redirected to /hi without any manual action
    FAIL if: redirects to /en or shows English content

    TEST 3 — I18N-03: Toggle switches locale and NEXT_LOCALE cookie persists
    1. Visit http://localhost:3000/hi/about
    2. Click "EN" in the header toggle
    3. Verify: URL changed to http://localhost:3000/en/about (not /en/hi/about or /hi/about)
    4. Open DevTools → Application → Cookies → find NEXT_LOCALE cookie → value should be "en"
    5. Navigate to http://localhost:3000/en/departments — verify still in English
    6. Click "HI" toggle → URL becomes /hi/departments
    PASS if: toggle navigates to same page in other locale; cookie reflects new locale
    FAIL if: URL is wrong (double prefix, wrong path) or cookie not updated

    TEST 4 — I18N-04: Nav links work correctly in both locales
    1. On http://localhost:3000/hi, click each nav link: होम, हमारे बारे में, विभाग, हमारे डॉक्टर, सेवाएं, संपर्क
    2. Verify: each URL has /hi/ prefix (e.g., /hi/about, /hi/departments) — NOT /about or /hi/hi/about
    3. Switch to EN via toggle, then click each nav link again
    4. Verify: each URL has /en/ prefix (e.g., /en/about, /en/departments)
    PASS if: all nav links include locale prefix; no 404s; no double prefix
    FAIL if: any nav link navigates to a non-prefixed URL or double-prefix URL

    TEST 5 — I18N-05: Devanagari script renders legibly
    1. Visit http://localhost:3000/hi
    2. Inspect any Hindi text visually — should show Devanagari script, not squares (tofu)
    3. Zoom to 200% — check that matras (diacritics like ि, ी, ा) are not clipped or overlapping
    4. On DevTools → Elements → inspect a Hindi text node → computed font-family should include 'Noto Sans Devanagari' or 'Nirmala UI' (depending on your OS) in the fallback chain
    5. Body text should be at minimum 16px with line-height 1.5 (inspect via DevTools computed styles)
    PASS if: Devanagari text renders clearly without tofu or clipped matras
    FAIL if: squares/boxes visible, matras collide at normal zoom

    BONUS: Cookie persistence across tab reopens
    1. Toggle to EN, close the tab (not the browser)
    2. Open a new tab, visit http://localhost:3000
    3. Should land on /en (NEXT_LOCALE session cookie still active)
    Note: This may not persist after full browser restart (session cookie) — that is expected per RESEARCH.md.
