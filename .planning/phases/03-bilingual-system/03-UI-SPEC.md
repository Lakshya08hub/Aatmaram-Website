| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, separator padding |
| sm | 8px | Compact spacing (toggle gap between HI and EN spans) |
| md | 16px | Default element spacing |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing (header height) |

**Phase 3 exception:** Touch targets on toggle spans must be at least 44px tall via `min-h-[44px]` or by inheriting height from the header row. The toggle renders inside the 64px header, so height is satisfied by container context. On mobile (drawer), the toggle slot in the drawer footer must have `min-h-[44px]` explicitly.

## Typography

Established in Phase 2, Phase 3 adds Devanagari to the font chain only — no size or weight changes.

| Role | Size | Weight | Line Height | Source |
|------|------|--------|-------------|--------|
| Body | 16px | 400 (regular) | 1.5 | Phase 2 established; `text-base` |
| Label / nav link | 14px | 600 (semibold) | 1.4 | Header nav: `text-sm font-semibold`; confirmed in `Header.tsx` |
| Heading | 20px | 600 (semibold) | 1.2 | Phase 2 established |
| Display | 28px | 600 (semibold) | 1.15 | Phase 2 established |

**Devanagari-specific:** Font chain for all body text after Phase 3 CSS change:
```
var(--font-geist-sans), 'Noto Sans Devanagari', 'Nirmala UI', system-ui
```
Geist handles Latin codepoints per-character; the next available font handles Devanagari codepoints. No JS involved — standard CSS Unicode range fallback.

**Devanagari legibility floor:** Body Hindi text must render at minimum 16px and line-height 1.5. Devanagari has taller ascenders than Latin — line-height below 1.5 causes matras (diacritics) to collide across lines.

| Role | Value | Usage | Source |
|------|-------|-------|--------|

| Secondary (30%) | `#1E3A5F` (`--color-nav`) | Header nav, sticky bar | `globals.css` → `--color-nav`; `Header.tsx` `bg-[#1E3A5F]` |
| Accent (10%) | `#16A34A` (`--color-accent`) | Book Appointment CTA only | `globals.css` → `--color-accent`; `Header.tsx` `bg-green-600` |
| Destructive | `oklch(0.577 0.245 27.325)` (`--destructive`) | Destructive actions only | shadcn default; no destructive actions in Phase 3 |
| Primary blue | `#1E40AF` (`--color-primary`) | Links, focus rings | `globals.css` project token |

**Accent reserved for:** Book Appointment CTA button only. The language toggle must NOT use the accent color — it is plain text on the nav background.

### Toggle Color States (Phase 3 only)

These are not new color tokens — they are opacity variants of white on the `#1E3A5F` nav background:

| State | Class | Computed appearance |
|-------|-------|---------------------|
| Active locale (current, non-clickable) | `text-white` | Full white — indicates "you are here" |
| Inactive locale (other, clickable) | `text-white/50` | 50% opacity white — indicates "click to switch" |
| Inactive locale hover | `text-white/80` | 80% opacity white — hover feedback |
| Separator pipe ` | ` | `text-white/30` | Recedes visually, not actionable |

**Rationale for active=bright, inactive=muted:** The muted state signals interactivity (something to click) without competing with the CTA. The bright state anchors the user's current context.

## LanguageToggle: Visual Specification

### Desktop (inside `Header.tsx` right-side action row)

```
[  HI  |  EN  ]   [Book Appointment]
```

- Layout: `flex items-center gap-1`
- Font: `text-sm font-medium` (14px, weight 500)
- Separator: plain `<span>` with `|` character, `text-white/30`
- Current locale span: `text-white` — not clickable, no cursor change, no underline
- Other locale span: `text-white/50 cursor-pointer hover:text-white/80 transition-colors`
- Keyboard: `role="button"` + `tabIndex={0}` + `onKeyDown` Enter handler on the clickable span
- No border, no background, no padding beyond the gap

### Mobile (inside `MobileNav.tsx` sheet drawer)

Placement: below the Book Appointment CTA in the drawer footer section, inside the `border-t` separator area.

```
[  Book Appointment  ]
─────────────────────
[  HI  |  EN  ]
```

- Layout: `flex items-center justify-center gap-2 py-3`
- Font: `text-sm font-medium`
- Colors: same opacity system BUT on white drawer background (`bg-white`): use `text-slate-400` for muted/inactive and `text-slate-700` for active
- Touch target: `min-h-[44px]` on the wrapping div for the toggle row

**Mobile toggle color override** (drawer is white, not navy):

| State | Desktop class | Mobile class |
|-------|--------------|--------------|
| Active locale | `text-white` | `text-slate-700` |
| Inactive locale | `text-white/50` | `text-slate-400` |
| Inactive hover | `text-white/80` | `text-slate-600` |
| Separator | `text-white/30` | `text-slate-300` |

The `LanguageToggle` component accepts an optional `variant` prop: `"nav"` (default, navy background) or `"drawer"` (white background). This avoids two separate components.

## Copywriting Contract

Phase 3 introduces no new page-level copy. The UI surface is the language toggle only.

| Element | English copy | Hindi copy | Notes |
|---------|-------------|-----------|-------|
| Toggle label — current locale (HI active) | `HI` | `HI` | Locale code, not translated |
| Toggle label — current locale (EN active) | `EN` | `EN` | Locale code, not translated |
| Toggle separator | ` | ` | ` | ` | Unicode pipe, same in both locales |
| Toggle aria-label (entire widget) | `"Switch language"` | `"भाषा बदलें"` | For screen reader announcement |
| Toggle aria-label (inactive span as button) | `"Switch to Hindi"` / `"Switch to English"` | `"हिंदी में देखें"` / `"अंग्रेज़ी में देखें"` | Describes the action, not the current state |
| Mobile hamburger button | `"Open navigation menu"` | (already aria-label, stays same) | Not user-visible copy |

**Translation register (all hi.json strings):**
- Formal `आप`-form throughout
- Natural Hindi, not literal word-for-word
- Proper nouns that stay in English: Ayushman Bharat, PM-JAY, ICU, OPD, Atmaram Child Care and Critical Care, Kanpur, Dr. [Name]
- Numbers stay as Arabic numerals (90 beds, 25+ doctors, 24x7)
- Medical terms: follow glossary in RESEARCH.md

**Primary CTA:** "Book Appointment" (EN) / "अपॉइंटमेंट बुक करें" (HI) — unchanged from Phase 2; translated in hi.json only.

**Empty state:** No new empty states introduced in Phase 3.

**Error state:** No new error states introduced in Phase 3. If a translation key is missing, next-intl renders `[missing "hi.namespace.key"]` — this is a developer-visible error, not a user-facing error. It must not appear in production (hi.json must be structurally complete).

**Destructive actions:** None in Phase 3.

## Interaction Contract

### Toggle Navigation Behavior

1. User clicks inactive locale span (or presses Enter on it)
2. `router.push(pathname, { locale: otherLocale })` fires — `pathname` from `usePathname()` is already locale-stripped
3. `syncLocaleCookie` sets `NEXT_LOCALE={locale}` in `document.cookie` immediately (client-side, before navigation)
4. Browser navigates to same path under new locale prefix (e.g. `/en/about`)
5. Page re-renders with new locale's translation strings
6. Toggle reflects new active locale

**Cookie persistence:** `NEXT_LOCALE` is a session cookie by default (no `maxAge`). Persists across tab reopens in the same browser session. Cleared on full browser close if "clear cookies on exit" is enabled. This matches D-03.

**Locale detection order on fresh visit:**
1. URL pathname locale prefix
2. `NEXT_LOCALE` cookie (if set from prior session in same browser session)
3. `Accept-Language` header (browser language preference)
4. `defaultLocale: 'hi'`

### Nav Link Behavior (after migration)

All `<Link>` components in `Header.tsx` and `MobileNav.tsx` import from `@/i18n/navigation`. The locale-aware `Link` prepends the current locale automatically. `href="/about"` renders as `/hi/about` or `/en/about` depending on context. No manual locale prefix in any href.

### Sheet Drawer Close on Navigation (Mobile)

`SheetClose render={<Link ... />}` pattern in `MobileNav.tsx` is preserved exactly. Only the `Link` import changes. Sheet closes on nav link click as before.

## Font Change Specification

**File:** `app/globals.css` — `@theme inline` block, line 9.

**Before:**
```css
--font-sans: var(--font-geist-sans);
```

**After:**
```css
--font-sans: var(--font-geist-sans), 'Noto Sans Devanagari', 'Nirmala UI', system-ui;
```

**Why this exact order:**
- `var(--font-geist-sans)` — handles Latin glyphs (set by `GeistSans.variable` on `<html>` in layout.tsx)
- `'Noto Sans Devanagari'` — Android 8+ built-in; clean sans-serif matching Geist aesthetic
- `'Nirmala UI'` — Windows 11 default Devanagari font (present when Hindi language pack installed)
- `system-ui` — iOS fallback (resolves to Kohinoor Devanagari or Devanagari Sangam MN)

No new CSS classes, no new `@font-face`, no CDN link, no npm package.

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Toggle keyboard operable | Inactive locale span has `role="button"`, `tabIndex={0}`, `onKeyDown` Enter handler |
| Toggle screen reader label | `aria-label="Switch to Hindi"` / `"Switch to English"` on the clickable span |
| Active locale not focusable | Active span has no `tabIndex`, no `role="button"` — it is presentational |
| Touch target size | Toggle row min-height 44px in mobile drawer; desktop inherits 64px header height |
| Color contrast (nav background `#1E3A5F`) | `text-white` on `#1E3A5F`: contrast ratio ~9.7:1 (WCAG AAA). `text-white/50` on `#1E3A5F`: ~4.2:1 (WCAG AA) |
| Devanagari legibility | min 16px body, 1.5 line-height; diacritic collision prevented |
