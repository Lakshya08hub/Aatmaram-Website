## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Visitor browser → /api/chat | User input is untrusted; client sends messages array |
| ChatWidget → public layout | Widget is a Client Component in a Server Component layout — no secrets cross this boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-05 | Information Disclosure | ChatWidget.tsx | mitigate | Widget never imports @google/genai or reads GEMINI_API_KEY; all Gemini calls go to /api/chat Route Handler |
| T-11-06 | Denial of Service (UI) | ChatWidget.tsx | accept | No server-side rate limiting on /api/chat for v1; widget is a public info tool with low abuse incentive; rate limiting can be added in Phase 12 if needed |
| T-11-07 | Tampering (XSS) | ChatWidget.tsx | accept | message.content is rendered as a text node (not dangerouslySetInnerHTML); React escapes HTML entities automatically — no XSS vector |
| T-11-08 | Usability (mobile overlap) | ChatWidget.tsx | mitigate | Bubble uses right-12 (48px) and bottom-8 (32px) offsets; panel uses max-w-[calc(100vw-48px)] to prevent overflow on small viewports |
</threat_model>
