## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 12px | 400 (regular) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 28px | 600 (semibold) | 1.2 |

Notes:
- Body (14px/400) applies to table cell content: staff name, role, salary amount.
- Label (12px/400) applies to table column headers and badge text.
- Heading (20px/600) applies to the page title "Payroll" and the summary card value "₹X,XX,XXX".
- Display (28px/600) is reserved for the summary card total amount figure only.
- Month navigator label ("June 2026") uses Heading size (20px/600) — it is the primary navigation landmark.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` (oklch 1 0 0 light / 0.145 0 0 dark) | Page background, table row backgrounds |
| Secondary (30%) | `--secondary` / `--muted` (oklch 0.97 0 0 light) | Summary card background, table header row, sidebar |
| Accent (10%) | `--primary` (oklch 0.205 0 0 light / 0.922 0 0 dark) | "Mark as Paid" button fill, Paid badge background |
| Destructive | `--destructive` (shadcn default red) | Not used in this phase (no delete/undo actions in v1) |

Accent reserved for:
1. "Mark as Paid" primary action button (per-row CTA)
2. "Paid" status badge background
3. Active focus rings on interactive elements

All other interactive elements (prev/next nav arrows, table sort, links) use `--muted-foreground` or default ghost/outline variants — never accent.

Semantic additions:
- Unpaid badge: `--muted` background + `--muted-foreground` text (neutral, not alarming)
- Paid badge: `--primary` background + `--primary-foreground` text (positive confirmation)
- Warning state (null salary marked paid): `--warning` or amber toast — do not use destructive red

## Layout Contract

### Page Structure

```
/payroll
├── Page Header (flex row, space-between, items-center)
│   ├── Title: "Payroll" (Heading 20px/600)
│   └── Month Navigator (flex row, gap-sm)
│       ├── ChevronLeft Button (ghost, 48px min-height, aria-label="Previous month")
│       ├── Month Label: "June 2026" (20px/600, min-w: 140px, text-center)
│       └── ChevronRight Button (ghost, 48px min-height, aria-label="Next month", disabled when selectedMonth >= currentMonth)
│
├── Summary Card (full width, muted background)
│   ├── Label: "Total Paid This Month" (12px/400, muted-foreground)
│   └── Value: "₹1,25,000" (28px/600, foreground) — sum of amount_paid for paid rows only
│
└── Staff Payment Table
    ├── TableHeader: Name | Role | Monthly Salary | Status | Action
    └── TableBody: one row per active staff member
        ├── Name (14px/400)
        ├── Role (14px/400, muted-foreground)
        ├── Salary (14px/400) — formatted as ₹X,XX,XXX or "—" if null
        ├── Badge: "Paid" (primary variant) | "Unpaid" (secondary/muted variant)
        └── Action cell:
            - Unpaid rows: "Mark as Paid" Button (default/primary, 14px, shows Loader2 during pending)
            - Paid rows: paid_at timestamp (12px/400, muted-foreground) e.g. "Paid on Jun 13"
```

### Month Navigator behavior
- Default: current calendar month on load
- Previous button: always enabled (no lower bound per D-05)
- Next button: disabled when `selectedMonth >= currentMonth` (prevents future-month payments)
- On navigate: table switches to skeleton loading state while Server Action re-fetches data

## Interaction States

### "Mark as Paid" button
| State | Visual |
|-------|--------|
| Idle | Primary variant Button, "Mark as Paid" label |
| Pending | Disabled, Loader2 spinner + "Marking..." label |
| Success | Button replaced by paid_at timestamp text; toast "Marked as paid — ₹X,XX,XXX" |
| Error (already paid) | Toast error "Already marked as paid for this month" |
| Error (server) | Toast error per copywriting contract below |
| Null salary warning | Success toast with warning: "Marked as ₹0. Update salary on the Staff page." |

### Month navigation
| State | Visual |
|-------|--------|
| Loading new month | Table rows replaced by Skeleton rows (same row count as previous month) |
| Future month blocked | ChevronRight Button disabled (opacity-50, cursor-not-allowed) |
| Empty month (no staff) | Empty state per copywriting contract |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | "Mark as Paid" |
| Page title | "Payroll" |
| Summary card label | "Total Paid This Month" |
| Summary card empty value | "₹0" (not "—" — zero is a valid and meaningful value) |
| Month navigator format | "June 2026" (full month name + 4-digit year, en-IN locale) |
| Table column: Name | "Name" |
| Table column: Role | "Role" |
| Table column: Salary | "Monthly Salary" |
| Table column: Status | "Status" |
| Table column: Action | "" (empty header — action column is self-evident) |
| Paid badge label | "Paid" |
| Unpaid badge label | "Unpaid" |
| Paid row action area | "Paid on [Day Mon]" e.g. "Paid on 13 Jun" |
| Null salary display | "—" (em dash, not zero — zero implies a set salary of ₹0) |
| Success toast | "Marked as paid — ₹[amount]" |
| Null salary success toast | "Marked as ₹0. Update salary on the Staff page first." |
| Already paid error toast | "Already marked as paid for this month." |
| Future month server error | "Cannot mark future months as paid." |
| Server error fallback | "Something went wrong. Please try again." |
| Empty state heading | "No active staff found" |
| Empty state body | "Add staff on the Staff page before running payroll." |
| Pending button label | "Marking..." |
| Disabled next button tooltip | "Cannot navigate to future months" |
| Previous month button aria-label | "Previous month" |
| Next month button aria-label | "Next month" |

Destructive actions: None in this phase. "Mark as Paid" is irreversible in v1 but is a positive action, not destructive. No confirmation dialog required. Irreversibility communicated via post-action toast ("This cannot be undone" is NOT needed — admin UX, not public).
