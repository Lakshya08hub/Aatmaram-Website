## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge internal padding |
| sm | 8px | Gap between stat chip label and value, table cell gap |
| md | 16px | CardContent internal padding, gap between stat chips |
| lg | 24px | Gap between sections (mt-6) |
| xl | 32px | Page padding (p-8 = 32px — matching payroll page) |
| 2xl | 48px | Top spacing between page header and first section (mt-8) |
| 3xl | 64px | Not used in this phase |

Exceptions: GA4 iframe height is 600px (fixed, not a spacing token). Touch targets on portal are not applicable — read-only Server Component with no interactive elements.

Source: payroll page pattern (`p-8`, `mt-8`, `mt-6` established in `PayrollClient.tsx`)

Source: PayrollClient.tsx established pattern; stat chip size is Claude's discretion (D-06)

  <Card className="mt-8">

    <CardContent>
      <iframe height="600" width="100%" ... />
      <p className="mt-2 text-xs text-muted-foreground">
        [helper text]
      </p>
    </CardContent>
  </Card>

  {/* Section 2: Appointment Operations */}
  <Card className="mt-6">
    <CardHeader>Appointment Operations</CardHeader>

  <Card className="mt-6">

### Dual Stat Chip Pattern (used in every section)

Two chips side by side inside `<div className="flex gap-4">`:

```
<div className="flex gap-4">
  <div className="flex-1 rounded-lg bg-muted p-4">
    <p className="text-xs text-muted-foreground">This Week</p>
    <p className="mt-1 text-2xl font-semibold">{weekCount}</p>
  </div>
  <div className="flex-1 rounded-lg bg-muted p-4">
    <p className="text-xs text-muted-foreground">This Month</p>
    <p className="mt-1 text-2xl font-semibold">{monthCount}</p>
  </div>
</div>
```

"This week" = rolling 7 days (today inclusive, 6 days prior). This is Claude's discretion (D-06 note in CONTEXT.md) — rolling 7 days is simpler than ISO week and avoids Monday-reset confusion for the admin.

### By-Status Breakdown (Appointment Operations only)

Three badges in a row below the dual stat chips, with `mt-4` gap:

```
<div className="mt-4 flex flex-wrap gap-2">
  <div className="flex items-center gap-2">
    <Badge variant="outline">Pending</Badge>
    <span className="text-sm font-semibold">{pendingCount}</span>
    <span className="text-xs text-muted-foreground">(this month)</span>
  </div>
  <div className="flex items-center gap-2">
    <Badge variant="default">Confirmed</Badge>
    <span className="text-sm font-semibold">{confirmedCount}</span>
    <span className="text-xs text-muted-foreground">(this month)</span>
  </div>
  <div className="flex items-center gap-2">
    <Badge variant="secondary">Cancelled</Badge>
    <span className="text-sm font-semibold">{cancelledCount}</span>
    <span className="text-xs text-muted-foreground">(this month)</span>
  </div>
</div>
```

### By-Doctor Table (Appointment Operations only)

Displayed below the by-status row, with `mt-4` gap. Uses shadcn Table components.

| Column | Width | Alignment |
|--------|-------|-----------|
| Doctor Name | auto (flex-fill) | left |
| This Week | 120px | right |
| This Month | 120px | right |

Table is not sortable — static read-only display. No pagination (portal has at most ~10 doctors). Empty rows show `—` for zero counts.

### Staff-by-Role Table (Staff & Payroll Summary only)

| Column | Width | Alignment |
|--------|-------|-----------|
| Role | auto | left |
| Active Staff | 120px | right |

Rows: one per distinct role in `profiles`. Maximum ~5 rows. No empty state needed (if no staff, show 0).

## Empty State Contract

Empty states appear inside the relevant section card, not as full-page replacements. Pattern:

```
<div className="py-8 text-center">
  <p className="font-medium">No data yet</p>
  <p className="mt-1 text-sm text-muted-foreground">
    No records found for the current period. Data will appear once activity is recorded.
  </p>
</div>
```

Applied when:
- By-doctor table has zero rows (no appointments this period)
- Staff-by-role table has zero rows (no active staff)
- Patient volume returns 0 for both periods (show 0 in chips, no separate empty state — zero is valid data)

## Interaction Contract

This is a pure Server Component. Interaction surface is minimal:

| Element | Interaction | Result |
|---------|-------------|--------|
| GA4 iframe | Browser-native iframe scroll/click | Handled by GA4 UI inside iframe |
| Page | Browser refresh (manual) | Re-fetches all stats from DB |
| All other elements | None | Static display only |

No hover states beyond Tailwind defaults. No focus traps. No modals. No form inputs.
