| Token | Value | Usage |

## Color

Inherited from established project design system.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | white / slate-50 | Page background, card backgrounds (`bg-white`) |
| Secondary (30%) | slate-200 / slate-100 | Card borders, table borders, dividers (`border-slate-200`) |
| Accent (10%) | blue-800 / blue-50 | Icon foreground in card icon circle (`text-blue-800`, `bg-blue-50`) |
| Destructive | red-600 / red-50 | Delete action buttons and confirmation dialogs only |

Accent reserved for: Department/Doctor/Facility card icon circles only. NOT used on section headings, grid containers, or portal table rows.

Featured indicator badge: Use `text-slate-500 text-sm` inline text — no accent color. Example: "3 of 8 featured".

Switch (featured toggle): Uses shadcn Switch default styling (blue when checked). No custom color override.

## Component Inventory

### New Component: FacilityCard

Pattern: Adapt DepartmentCard. Same visual structure — icon circle + name + description.

```
Props:
  name: string
  description: string          — truncate at 120 characters with "…" suffix
  image_url?: string | null    — if present, render as 48×48 rounded image instead of icon circle
```

Visual spec:
- Card: `bg-white border border-slate-200 rounded-xl shadow-sm hover:shadow-md transition-shadow`
- Image/icon container: `w-12 h-12 rounded-full` — image uses `object-cover`, fallback icon is `Building2` from lucide-react with `text-blue-800 bg-blue-50`
- Name: `text-xl font-semibold text-slate-900 mt-4`
- Description: `text-sm text-slate-500 mt-2 leading-relaxed` — clamped to 3 lines max via `line-clamp-3`

### Homepage — Facilities Section

Placement: After Doctors section, before Appointment CTA band.

Section structure:
```
<section className="py-16 bg-slate-50">
  <SectionHeading title="Our Facilities" />
  <div className="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-6 mt-8">
    {facilities.map(f => <FacilityCard ... />)}
  </div>
</section>
```

Grid breakpoints: 1 column mobile, 2 columns tablet (sm: 640px+), 3 columns desktop (lg: 1024px+). Gap: 24px (gap-6).

Focal point: The "Our Facilities" section heading (`SectionHeading`) is the visual entry point for the section. The FacilityCard name (`text-xl font-semibold`) is the secondary focal anchor within each card.

### Portal — Featured Toggle Column (all three content pages)

Added inline to existing Table in DepartmentsClient, DoctorsClient, FacilitiesClient.

New columns added after existing data columns, before the Actions column:

| Column header | Content | Width |
|---------------|---------|-------|
| Featured | shadcn Switch, checked = is_featured | w-24 (96px) |
| Order | `<input type="number" min="1" max="99">` | w-14 (56px) |

Featured count indicator: Rendered above the table as a `text-sm text-slate-500` badge:
- "N of M featured" — where N is count of is_featured=true rows, M is total rows
- No color highlight — plain slate text, right-aligned relative to the table header row

Switch behavior:
- On toggle: immediate Server Action call, optimistic UI update via local state, toast.success/toast.error on resolution
- Disabled state during pending action: `disabled` prop on Switch + `opacity-50 cursor-not-allowed`

Order input behavior:
- `type="number"` min=1, no stated max (accept any positive integer)
- Width: `w-14` (56px), `text-center`
- On blur: Server Action call if value changed, toast.success/toast.error on resolution
- No spinner needed for order input — fire-and-forget with error toast only

## Interaction States

### Homepage Sections (Departments, Doctors, Facilities)

| State | Visual |
|-------|--------|
| Loading (Server Component) | N/A — page renders fully server-side; no skeleton needed |
| Data available | Grid of cards |
| Fallback (no featured) | Same grid, shows all active records — no UI distinction from normal state |

### Portal Toggle Column

| State | Visual |
|-------|--------|
| Switch idle (off) | shadcn Switch unchecked default |
| Switch idle (on) | shadcn Switch checked (blue) |
| Switch pending | Switch disabled + opacity-50 |
| Switch success | toast.success "Saved" (1.5s) |
| Switch error | toast.error "Failed to save. Try again." |
| Order input idle | Plain number input |
| Order input blur (changed) | Fires Server Action silently; toast on error only |
| Order input error | toast.error "Failed to update order. Try again." |
