<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| .env.local → Next.js runtime | Secrets injected from the host filesystem into the Node.js process at startup |
| NEXT_PUBLIC_* vars → browser bundle | Any var prefixed NEXT_PUBLIC_ is embedded in the client JS bundle — visible to all users |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | .env.local | mitigate | .gitignore must contain .env.local before this file is created — verify grep -c ".env.local" .gitignore returns >= 1 before writing .env.local |
| T-02-02 | Information Disclosure | SUPABASE_SERVICE_ROLE_KEY | mitigate | Must NOT have NEXT_PUBLIC_ prefix — server-only; bypasses Row Level Security if leaked |
| T-02-03 | Information Disclosure | GEMINI_API_KEY | mitigate | Must NOT have NEXT_PUBLIC_ prefix — API calls go through app/api/ routes only (Phase 11) |
| T-02-04 | Elevation of Privilege | Supabase anon key | accept | Anon key is safe to expose (RLS controls access) — NEXT_PUBLIC_ prefix is intentional for this key only |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No new installs in this plan — only env config and connection test |
</threat_model>
